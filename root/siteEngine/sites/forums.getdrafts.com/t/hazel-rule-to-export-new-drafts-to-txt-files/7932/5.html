sylumer July 15, 2020, 6:23pm #2

Hazel works on files, but Drafts’ drafts are held in a database, not in files. The only way you could have Hazel do something like this is if you were to periodically export any new drafts to a folder. At that point all the normal rules of Hazel would apply and you could use many of the import automations available for Devon Think at that point.

miscellaneplans August 16, 2021, 9:50pm #6

I use Hazel to move files created by Drafts in the Drafts folder within iCloud, to another apps folder—also in iCloud. It’s simple, and I hope it helps you achieve what you’re looking for. This is a link to the article I wrote about it.

The second rule I use is for appending text to a file that I want to show up outside of the Drafts folder. It uses transclusion, so I don’t know if it’ll be helpful outside of Obsidian.

Drafts is set up to create then append text to a note (YYYY-MM-DD - Captain’s Log) in the Drafts folder in iCloud. Hazel watches that folder and creates a copy of the file to the Obsidian folder (also in iCloud) every time it is modified. I have a daily note set up in Obsidian with a transclusion using the same name as the file Drafts created, so that file shows up as part of my daily note and can be added to by Drafts throughout the day.
